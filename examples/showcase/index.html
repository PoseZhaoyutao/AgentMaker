团队分工

六个角色，一个目标守门人

🧭

Main Agent

维护宪章、目标、约束与验收，做最终审查。不是转发员，是目标守门人。

🔭

Product Scout

补足用户场景、使用者、机会点和默认假设。

🎨

Design Agent

把设计方向变成体验结构、页面、功能与规则。

🛡️

Constraint Agent

把约束边界变成硬规则、风险列表和不能碰的线。

🔨

Builder Agent

产出文件与可运行成果，优先通过已装技能构建。

✅

Verification Agent

跑测试与验收检查，失败就要求修正并给证据。

技能编排

不重造轮子，复用你已装的技能

产出文件前先读对应技能，再通过它构建。这就是 Fable 5「产出前先读 SKILL.md」的做法。

要做 PPT / 演示→pptx
要做 Word 文档→docx
要做表格 / 清洗数据→xlsx
要创建 / 填写 PDF→pdf
要做网页 / 组件 / 应用→frontend-design
领域任务（写作 / 配图 / 文献）→对应领域技能
